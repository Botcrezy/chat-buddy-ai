

# إصلاح تكرار رسالة الترحيب وتحسين جودة الردود

## المشكلة
البوت بيكرر نفس رسالة الترحيب مع كل رسالة من العميل. السبب من اللوجز:
1. أول محاولة (9 رسائل + system prompt طويل) ← الموديل يرجع **فاضي**
2. Retry بـ 2 رسالة بس (system prompt + آخر رسالة) ← **يشتغل** لكن بدون تاريخ المحادثة، فيفتكر إنها محادثة جديدة ويبعت ترحيب

## الحل في `supabase/functions/whatsapp-webhook/index.ts`

### 1. تقصير System Prompt بشكل جذري
- حذف بيانات التدريب الكاملة من الـ system prompt
- إرسال بيانات التدريب المرتبطة فقط (مطابقة كلمات الرسالة)
- تقليل الحد الأقصى من 4000+1500 حرف إلى 2000 حرف كحد أقصى

### 2. إصلاح منطق الـ Retry
- بدل إرسال system + آخر رسالة فقط، إرسال system + آخر 4 رسائل
- إضافة تعليمة صريحة: "لو العميل قال كلام قصير زي اوك/تمام/طيب، ردي عليه طبيعي متقوليش أهلاً تاني"

### 3. إضافة منطق ذكي للرسائل القصيرة
- لو الرسالة أقل من 10 حروف (زي "اوك"، "تمم"، "طيب") + فيه محادثة سابقة → إضافة سياق "العميل بيكمل محادثة مش بيبدأ جديدة"

### 4. تحسين مطابقة بيانات التدريب
- بدل إرسال كل الـ knowledge_base، عمل بحث بالكلمات المفتاحية في رسالة العميل
- إرسال أقرب 5-10 نتائج فقط
