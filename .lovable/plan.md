

# خطة تطوير شاملة: دعم الصوت + الصور + البحث + تدريب ضخم

## ملخص

1. دعم فهم الرسائل الصوتية (Voice Notes) من العملاء
2. دعم فهم الصور المرسلة من العملاء (Image Understanding)
3. إضافة بحث عبر الإنترنت للبوت
4. تحسين ذكاء البوت (عدم تكرار اسم العميل + تعامل بشري)
5. إضافة 150+ بيان تدريبي من بيانات Sity Cloud الكاملة

---

## الجزء الأول: دعم الصوت (Voice Notes)

### المشكلة
حالياً الـ Baileys Server يستقبل `audioMessage` لكن يبعت فقط `mediaType: 'audio'` بدون محتوى. الـ webhook يتجاهل الصوت.

### الحل
1. **Baileys Server**: تحميل الصوت كـ buffer → تحويله لـ base64 → إرساله مع الـ webhook
2. **Webhook**: استخدام OpenRouter مع نموذج يدعم audio transcription أو استخدام Google Speech-to-Text المجاني عبر Whisper model

**ملاحظة تقنية**: النماذج المجانية على OpenRouter لا تدعم transcription مباشرة. الحل: استخدام Whisper API المجانية من Groq أو استخدام نموذج يدعم multimodal. بما إن المشروع ملتزم بالمجاني، سنحول الصوت لنص باستخدام OpenRouter model يدعم audio input.

**الحل العملي**: في الـ baileys-server، نحمّل الفويس نوت كـ base64 ونبعتها مع الـ webhook. في الـ webhook، نستخدم `google/gemma-4-31b-it:free` (يدعم multimodal) أو نستخدم Groq Whisper المجاني لتحويل الصوت لنص.

### الملفات المتأثرة
- `baileys-server/index.js` - تحميل الصوت كـ base64
- `supabase/functions/whatsapp-webhook/index.ts` - معالجة الصوت

## الجزء الثاني: فهم الصور من العملاء

### المشكلة
حالياً الصور بتتحمّل كـ base64 لكن الـ webhook مش بيفهم محتواها.

### الحل
إرسال الصورة كـ base64 في رسالة multimodal للنموذج عبر OpenRouter. النماذج الحرة زي `google/gemma-4-31b-it:free` تدعم فهم الصور.

### التغييرات
- `supabase/functions/whatsapp-webhook/index.ts` - إضافة الصورة في الـ chat messages كـ image_url content

---

## الجزء الثالث: البحث عبر الإنترنت

### الحل
استخدام DuckDuckGo HTML API (مجاني بالكامل) للبحث عن معلومات. البوت يبحث لما الموضوع يكون خارج بيانات التدريب ومحتاج معلومة حقيقية.

### التنفيذ
- إضافة function في الـ webhook تعمل web search
- البوت يقرر إذا كان محتاج بحث بناءً على السياق
- النتائج تتضاف للـ context

---

## الجزء الرابع: تحسين ذكاء البوت

### التحسينات في الـ System Prompt
1. **عدم تكرار اسم العميل**: تعليمات واضحة إن البوت يستخدم الاسم مرة واحدة في البداية فقط
2. **تعامل بشري**: تحسين التعليمات للتعامل الطبيعي
3. **استخدام الـ memory بذكاء**: الرجوع لذاكرة العميل وتاريخه بشكل سلس
4. **ردود أقصر وأدق**: تعليمات أوضح
